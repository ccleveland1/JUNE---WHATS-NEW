What's New
Version History
New features, enhancements and bug fixes.
Version OAN 20-03.0.0
People Work History – Add “Create People Record” ActionNew Feature
A new feature has been added to the People Record Work History screen to allow the creation of new People Records, based on information parsed from the Work History record. The action is located on the left menu, as shown:
People Screen Required Fields ReductionEnhancement
Many OANet users have tried to create a People Record only to find they lacked sufficient information to fill all required fields. In order to allow users to quickly create People without requiring the inclusion of unnecessary details, the number of required fields needed to create a People Record has been reduced and a new dynamic system introduced that only requires additional fields for specific actions. By reducing the number of required fields, AEs now have more options for nurturing candidates.
For further information on the reduction of fields in the People Screen, please visit the Release Notes.
Save SMS to Activity Comments Enhancement
The functionality allowing users to save SMS messages to Activity Comments has been updated for ease of use. Users will now be able to access a given People Record when clicking on the respective name from within the SMS messenger; users will also have the ability to show/hide selection checkboxes, and can now identify messages already saved to Activity Comments by the presence of a highly visible checkmark next to the message timestamp.
Skip APPS Validation for Vendor PlacementsEnhancement
When creating a new Vendor Placement, users will no longer be require to submit APPS validation (as APPS data is not collected for Vendor workers/ICs)
Delaware Sexual Harassment Prevention TrainingEnhancement
Per state mandated requirements, all staff that have a report-to state of Delaware will now be required to complete a one-hour sexual harassment prevention training course when they begin their assignment. Once they complete this training, they will not require training again for two years. When the temp completes the training, OANet will automatically upload the certificate of training completion to their people record.
Job to Fill Analysis (JFA) Report – “Zip Code Radius” Report Criterion AddedEnhancement
The ability to choose Zip Code Radius, based on the Client’s address, has been added to the JFA Report generation criteria. This will filter the report output to include only Job Orders which have Clients whose addresses fall within the specified Zip Code Radius. This new criterion can be found on the JFA Report Criteria screen as follows:
Client Lookup – Fields Moved to Main SearchEnhancement
When performing a Client Lookup search, the following fields will now be available in the main search criteria (where they were previously only available when using the Advanced Search functionality):
  • Name
  • Client ID
  • Corporate ID
  • Creation Date
Job Order Recommendation – Prevent DuplicatesEnhancement
OANet will now automatically prevent the addition of duplicate candidates to Job Order Recommendation and will verify ensure Recommendations for a given Job Order are unique using candidate SSN and/or email address.
Import Job Ad Response Without Physical ResumeEnhancement
When importing a Job Ad Response and a People Record for the respective candidate already exists, users might encounter a "no resume found" error if physical resume is missing. In these instances, OANet will no longer error but instead import resume from the resume text section of the existing People Record.
Copying Records – Functionality UpdateEnhancement
While the ability to copy records has long been a feature of OANet, the functionality has been updated to clarify what items will be copied in the copy record being created. A number of previously ommitted record details are now included when copying Job Order and Placement records, including Job Description, Private Notes, Start Date, End Date, and Actual End Date. For copied People Records, Status will now be shown, and fields not being copied will be more clearly indicated. When copying Client records, users will now have the option to copy the Credit Screen as well. It is not possible to copy Leads People or Leads Client records
Activity Report – Limit NotesEnhancement
A checkbox has now been added to the Activity Report builder allowing users to limit notes to 1000 characters. This will limit the length of notes and Activity Comments in the report output to 1000 characters, as long notes occasionally make the process of building/reading reports highly cumbersome.
Employee Hour ReportEnhancement
The People Hour Sheet Report and Client hour Sheet Report have received a number of updates – new columns have been added to show Pay Code, Bill amount, Non Bill Amount (beside other info). Users may now choose one or more pay codes from the screen, and have additionally been provided the option to display bill amount and non-bill amount.
Action Buttons at Top and Bottom of PagesEnhancement
Action buttons such as "Submit", "Continue", "Cancel", "Clear", etc. can now be found on the top right of relevant pages, in addition to their original location at the bottom center of the page. Buttons like "Submit", "Continue", "Save" now have a blue background and white type. Buttons like "Cancel", "Clear", "Delete" have a grey background and black type. The Refresh button is now an icon
Alert Section – Blurry TextBug Fix
Users were previously experiencing an issue with blurry text in the Alerts section. This issue has been addressed and the text should now appear clearly for all users, with a larger font size and contrasting background ensuring clarity. The alert icons have also been updated
People Management Report – Origination Source As Default People SourceBug Fix
In keeping with sales initiatives devised by the Talent Development department, the PMR will now run with People Source set to Origination Source by default – the report criterion previously defaulted to Recruiting Source
Manual Time Card Entry – Pay Code FieldBug Fix
While manually entering Time Cards, long Pay Codes could be hard to read in their entirety due to the small size of the Pay Code field. This field has been expanded to allow users to read long Pay Codes
